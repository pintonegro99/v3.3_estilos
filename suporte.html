13.Browser suportado

Embora não tenha sido construído para os browsers mais antigos, a estrutrua original da framework "spectre" tem suporte para algumas versões antigas. No entanto para uma performance ideal recomendada-se a utilização da versão mais recente do browser.

A listagem do suporte ao browser é a seguinte:

  • Chrome (últimas 4 versões)
  • Microsoft Edge (últimas 4 versões)
  • Firefox (suporta qualquer versão)
  • Safari (últimas 4 versões)
  • Opera (últimas 4 versões)
  • Internet Explorer 10+ (browser desatualizado, recorrer ao Edge) *

*É suportado no Internet Explorer 10+, mas alguns recursos de HTML5 e CSS3 não são perfeitamente interpretados pelo browser do Internet Explorer.

15.Os temas

Os temas web trabalhados para a Fundação têm uma base de construção inspirada na "Framework da Bootstrap". O tema estilos foi trabalhado segundo alguns critérios de aparência visual, provenientes da maqueta e do desenho inicial para as soluções de User Interface.

Uma estrutura modelo

A estrutura original de construção é inspirada na framework "Spectre". Esta base não pode ser mexida, e a CSS original deve ser respeitada, mas qualquer tema personalizado ou a personalizar pode ser trabalhado, alterado e carregado. No entanto deve estar inserido num esquema semelhante ao que indicamos ao lado.


        ├── Nome do projeto/
        │   ├── dist/                       
        │   │   └── base.spectre.css            
        │   │
        |   ├── css/                       
        │   │   └── guia.css
        │   │   └── estilos.themeFCT.css (o tema) 
        │   │   └── estilos.themeFCT.icones.css 
        │   │   └── fonts/  
        │   │     └──── FCTicones.eot;FCTicones.svg;  
        │   │           FCTicones.ttf;FCTicones.woff 
        │   │    
        |   ├── docs/
        │   │   └── documento_um.pdf
        |   ├── img/
        │       └── imagem_um.jpg

16.O projeto estilos, no github...

O Git é um sistema gratuito para controle de versões. Controlar versões significa gerir diferentes versões dos projetos em arquivo, com a possibilidade de recuperar as versões salvas, reverter a um projeto inicial, ou poder comparar as versões de quem programa e desenvolve projetos web em simultâneo. Isto é uma mais valia para qualquer programador ou designer e por isso investir tempo nesta temática é angariar boas e melhores práticas de trabalho. Pela facilidade e por existir software livre também já estamos no GitHub. O download do projeto no Git com todo o material necessária pode ser acedido por aqui.

17.Atualizações

O projeto estilos tem muitas horas de trabalho, tal como a anterior styleguide, dedicadas a reunir e a apresentar melhor a informação. As bases de inspiração são as estruturas web com boas referencias, apresentação dos conteúdos, e sobretudo as soluções visuais para os elementos e as componentes de UI.
